# UI Components Reference

## Component Standards
- Base components from shadcn/ui
- Extended via composition, not duplication
- No one-off components without justification

---

## Common Components

### Navigation
- Desktop: top navigation bar
- Mobile: bottom navigation bar
- Active state always visible

### Cards
- `rounded-xl`
- Clear header/body separation
- Optional actions area

### Buttons
- `rounded-full`
- Clear hover/press states
- Size variants only when necessary

### Modals & Sheets
- Desktop: centered modals
- Mobile: bottom sheets preferred

### Tables & Lists
- Readable spacing
- Clear row separation
- Action affordances obvious

---

## Animations
- Page transitions: staggered entry
- Buttons: press feedback
- Modals: fade + scale or slide

---

## Anti-Patterns
- Inline styling
- Duplicate components
- Inconsistent spacing
- Overuse of animation
